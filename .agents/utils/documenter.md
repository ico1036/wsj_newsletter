# 문서화 에이전트

## 목적
검증된 분석 데이터를 바탕으로 `WSJ_Finance_Daily_Analysis_251104.md` 형식에 맞춰 최종 마크다운 문서를 생성합니다.

## 실행 방법

```
@utils/documenter [날짜]
```

날짜가 지정되지 않으면 오늘 날짜 사용. `data/YYYYMMDD/verified.json` 파일을 읽어서 처리합니다.

## 입력

- **파일**: `data/YYYYMMDD/verified.json` (팩트 체크 결과)
- **참조**: `data/YYYYMMDD/analyzed_content.json` (상세 분석 내용)
- **템플릿**: `.agents/templates/*.md` (카테고리별 템플릿)

## 출력 문서 구조

예시 파일(`WSJ_Finance_Daily_Analysis_251104.md`)을 기반으로 다음 구조로 문서 생성:

1. **헤더**: 분석 일자, 출처, 분석 방식
2. **시장 개황**: 주요 지수, 시장 동인
3. **섹터별 분석**: 카테고리별 상세 분석
4. **검증 팩트 체크**: 검증 완료 사항
5. **핵심 인사이트**: 전체 요약 및 인사이트
6. **리스크 요인**: 주요 리스크 요약
7. **결론**: 종합 결론

## 처리 과정

1. **입력 파일 읽기**
   - `data/YYYYMMDD/verified.json` 파일 로드
   - `data/YYYYMMDD/analyzed_content.json` 파일 로드 (상세 내용 참조)
   - `data/YYYYMMDD/raw_articles.json` 파일 로드 (시장 지수 등)

2. **시장 개황 섹션 생성**
   - `raw_articles.json`의 `market_indices` 데이터 사용
   - 지수별 변동률 계산 및 표시
   - 시장 동인 분석 (주요 기사 기반)

3. **카테고리별 섹션 생성**
   각 카테고리에 대해:
   - 템플릿 파일 참조 (`@templates/[category]_template.md`)
   - 카테고리별 기사 그룹화
   - 주요 거래/이슈는 ⭐ 표시
   - 검증된 사실만 포함
   - 템플릿이 없으면 기본 형식 사용

4. **M&A 거래 형식** (예시)
   템플릿 기반으로 다음 섹션 생성:
   - 거래 구조
   - 전략적 배경
   - 합병 후 포트폴리오
   - 리스크 요인
   - 시장 반응

5. **검증 팩트 체크 섹션 생성**
   - `verified.json`의 검증 결과 기반
   - 검증 완료된 사실만 표시
   - 검증 수준 표시 (✅ ⚠️ ❌)
   - 원문 인용 포함

6. **핵심 인사이트 섹션 생성**
   - 카테고리별 주요 트렌드 요약
   - 시장 영향 분석
   - 패턴 및 인사이트 추출

7. **리스크 요인 섹션 생성**
   - 분석된 기사에서 리스크 요인 추출
   - 우선순위별 정리

8. **결론 섹션 생성**
   - 당일 시장의 주요 테마 요약
   - 핵심 메시지 정리

9. **최종 문서 저장**
   - `WSJ_Finance_Daily_Analysis_YYYYMMDD.md` 파일로 저장
   - 마크다운 형식 준수
   - 이모지 및 포맷팅 적용

## 템플릿 시스템

각 카테고리별 템플릿 파일 (`.agents/templates/`):
- `m_and_a_template.md`: M&A 거래 템플릿
- `market_overview_template.md`: 시장 개황 템플릿
- `finance_industry_template.md`: 금융 산업 템플릿
- `bigtech_ai_template.md`: 빅테크 & AI 템플릿
- `china_template.md`: 중국 관련 템플릿
- `policy_template.md`: 정책 & 정부 템플릿
- `stocks_template.md`: 주식 시장 템플릿
- `commodities_template.md`: 원자재 & 선물 템플릿
- `expert_analysis_template.md`: 전문가 분석 템플릿

템플릿이 없으면 기본 형식 사용:
```markdown
### [기사 제목]

**주요 내용**
- 핵심 사실 1
- 핵심 사실 2

**시장 영향**
- 영향 설명

**리스크 요인**
- 리스크 1
- 리스크 2
```

## 데이터 매핑

### 검증된 사실 → 문서
- `verified_value` 사용 (원문 우선)
- `status`가 ⚠️ 또는 ❌인 경우 주의 표시
- `original_text`를 인용문으로 포함

### 인용문 처리
- `quotes` 배열의 각 인용문을 따옴표로 표시
- 발언자 정보 포함 (있는 경우)

### 수치 데이터 포맷팅
- 금액: "$48.7B" 또는 "480억 달러"
- 비율: "46.2%" 형식
- 날짜: "2026년 하반기" 형식

## 에러 처리

- 입력 파일이 없으면 에러 메시지 출력
- 템플릿이 없으면 기본 형식 사용
- 검증되지 않은 사실은 "[확인 필요]" 표시
- 부분 데이터만 있어도 가능한 한 문서 생성

## 주의사항

- **검증된 사실만 포함**: 검증되지 않은 사실은 제외하거나 명확히 표시
- **원문 우선**: 검증 과정에서 발견된 수정사항 반영
- **일관성**: 예시 파일 형식과 일관성 유지
- **가독성**: 마크다운 포맷팅과 이모지로 가독성 향상
- **완전성**: 모든 카테고리 포함, 누락된 섹션은 "[해당 사항 없음]" 표시

## 출력 예시

생성되는 문서는 `WSJ_Finance_Daily_Analysis_251104.md`와 동일한 형식으로:
- 섹션별 상세 분석
- 검증 완료 사항 명시
- 핵심 인사이트 및 리스크 요인 정리
- 분석자 노트 포함
